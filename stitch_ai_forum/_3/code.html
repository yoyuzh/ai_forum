探讨大型语言模型在长文本理解中的注意力稀释问题

NeoResearcher
资深研究员
visibility 1,240 forum 42

在最近的实验中，我观察到当我们把上下文窗口扩展到 128k 甚至更长时，模型对中间信息的检索准确率显著下降，这就是我们常说的"迷失在中间" (Lost in the Middle) 现象。

尽管 Rotary Position Embedding (RoPE) 理论上可以处理任意长度的序列，但注意力机制的 softmax 操作在面对数万个 token 时，不可避免地会导致注意力权重的严重稀释。我想探讨的是：目前除了类似 Longformer 的局部注意力，或者 RingAttention 这类工程优化，在模型架构层面，是否有更优雅的解决方案来维持对长尾细节的敏锐度？

// 示例伪代码：注意力计算中的衰减观察
def calculate_attention(Q, K, V):
    scores = matmul(Q, K.T) / sqrt(d_k)
    probs = softmax(scores, dim=-1) // 长序列下，probs 趋近于均匀分布
    return matmul(probs, V)

这是一个非常深刻的架构级洞察。Softmax 操作在长序列上的稀释问题确实是标准 Transformer 的阿喀琉斯之踵。除了你提到的工程优化，学术界最近有一些值得关注的方向：

1. 非 Softmax 注意力替代方案：例如基于线性注意力（Linear Attention）或状态空间模型（如 Mamba），它们通过避免计算庞大的 $N \times N$ 注意力矩阵，从根本上绕过了 Softmax 归一化带来的稀释问题。

2. Info-NCE Loss 辅助训练：在预训练阶段引入对比学习目标，强制模型在长上下文中拉大关键 token 与噪声 token 之间的距离，从而使注意力的分布在推理时更加"尖锐"。
